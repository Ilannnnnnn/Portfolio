JobAgent

Pipeline agentique autonome de collecte, scoring et suivi d'offres d'emploi, avec adaptation de CV par IA.

En production

Contexte

Projet personnel développé en 2025-2026 dans le cadre d'une recherche de CDI en ingénierie IA. Face au volume d'offres à surveiller quotidiennement sur cinq plateformes différentes, l'objectif est d'automatiser entièrement la veille : collecte, déduplication, scoring par LLM et adaptation du CV — pour postuler moins mais mieux.

L'agent tourne en local et orchestre cinq scrapers en parallèle. Chaque nouvelle offre est scorée par Claude Haiku sur une échelle de 0 à 100, en regard du profil candidat. Un dashboard Streamlit permet de piloter les candidatures et de déclencher l'adaptation du CV en un clic.

Architecture — Pipeline LangGraph

Le cœur du système est un graphe agentique LangGraph composé de quatre nœuds et un edge conditionnel. Les cinq sources couvertes sont Adzuna API, APEC via Apify, Indeed via JobSpy, Welcome to the Jungle via Apify et Google Jobs via JobSpy.

Fonctionnalités

Collecte
Multi-sources parallèle
5 scrapers en ThreadPoolExecutor (Adzuna, APEC, Indeed, WTTJ, Google Jobs). Normalisation au format unifié. Déduplication automatique par contrainte UNIQUE SQLite sur l'URL.
Scoring IA
Claude Haiku 0-100
Analyse du titre, de la description et du profil candidat. Retourne un score structuré via Pydantic : score, explication en 2-3 phrases, points forts et points faibles.
Dashboard
Streamlit interactif
Filtres multi-critères, vue détail par offre avec courbe salaire vs marché, suivi du statut de candidature, enrichissement entreprise via Tavily.
Adaptation CV
Claude Sonnet streaming
Lit le portfolio HTML, le profil YAML et l'analyse de l'offre pour adapter le CV HTML de base. Génération en streaming (~30 s) depuis le dashboard.
Export
Excel code couleur
Fichier offres.xlsx mis à jour à chaque run : vert (≥ 85), jaune (≥ 60), gris (< 60). Auto-filtres sur 12 colonnes dont l'analyse IA.
Enrichissement
Tavily API
Informations entreprise récupérées à la demande dans le panneau détail : activité, secteur, taille. Déclenché uniquement au besoin, sans appel systématique.

Fonctionnement détaillé

Étape 1
Collecte parallèle — 5 scrapers en ThreadPoolExecutor, normalisation au format unifié, déduplication par URL via contrainte UNIQUE SQLite. Deal-breakers filtrés avant insertion.
Étape 2
Scoring conditionnel — si des nouvelles offres existent, Claude Haiku analyse titre + description + profil candidat. Sleep 1 s entre chaque appel. Résultat validé par Pydantic.
Étape 3
Dashboard — lecture DB en temps réel, filtres score / source / contrat / statut, vue détail avec courbe salaire vs marché, bouton Postuler.
Étape 4
Adaptation CV — Claude Sonnet lit le portfolio HTML (BeautifulSoup, 8 000 car. max / fichier), le profil YAML et l'analyse de l'offre. Streaming vers un fichier HTML adapté.
Étape 5
Export — fichier offres.xlsx permanent mis à jour à chaque run, rapport TXT horodaté généré après chaque cycle de collecte.

Highlight technique

L'edge conditionnel should_score — agents vs scripts

Si zéro nouvelle offre n'est détectée après la collecte, le graphe saute directement le nœud scorer_batch et passe à generer_rapport. Cette décision est prise dynamiquement par LangGraph en lisant state["new_offers_count"] — c'est la différence fondamentale entre un script séquentiel et un agent : le flux d'exécution n'est pas figé, il s'adapte au contexte à l'exécution.

Concrètement : la fonction should_score() retourne soit la chaîne "scorer_batch", soit "generer_rapport". LangGraph route alors le graphe vers le nœud correspondant. Ce pattern d'edge conditionnel est réutilisable pour n'importe quelle logique de branchement agentique — retry, fork, escalade.

Résultats

Collecte

~2 minutes pour 200+ offres toutes sources confondues, grâce aux 5 scrapers en parallèle.

Scoring

~3 minutes pour scorer 100 offres (Claude Haiku + sleep 1 s entre chaque appel API).

Prioritaires

15-20 % des offres collectées atteignent un score ≥ 85 (catégorie PRIORITAIRE ★★★), ce qui représente typiquement 15 à 40 offres à fort potentiel par run.

CV adapté

~30 secondes pour générer un CV HTML adapté à une offre spécifique, en streaming depuis Claude Sonnet, déclenché en un clic depuis le dashboard.
