Assignments & Projects

Five concurrent assignments covering the full AI-applied-to-telecom-infrastructure chain.

Assignment 1
AI assistant for PCAP trace analysis — Development of an assistant capable of automatically analysing network capture files (PCAP/Wireshark). The central challenge: raw PCAP files have entropy close to the theoretical maximum (~7.5 bits/byte), making direct LLM processing very difficult. Exploration of multiple approaches: RLM, LLMcap, TrafficLLM, entropy-based segmentation inspired by the Byte Latent Transformer. Use of Orange's internal LLM proxy (Gemini 2.5 Flash Lite). Key insight: extracted CSV formats (CIC/UNB-style) are more exploitable than raw binary. This work is part of an agentic AI initiative applied to telecom technologies, currently undergoing commercial valorisation.
Assignment 2
Qwen2.5-3B fine-tuning for telecom support — Supervised fine-tuning of a 3-billion-parameter LLM for technical telecom support question answering. Systematic evaluation of 16 variants against a structured rubric (relevance, solution quality, completeness, clarity, absence of errors — 10 pts). Result: no variant outperformed the base model (~88/90 pts). Issues identified: question repetition, text duplication, hallucinations, topic drift. Concrete learning: a fine-tuned model does not automatically outperform the base model — rigorous evaluation is essential before deployment.
Assignment 3
IPSEC gateway in Go (A-SBC/IPSECGW) — Development and maintenance of an IPSEC/SIP gateway in Go handling up to 250,000 simultaneous users. Two focus areas: (1) Cryptographic migration from MD5 to SHA-1-96 for ESP IPSEC, with a critical subtlety — the A-SBC server accepts SHA-1 for IPSEC but announces MD5 for SIP digest authentication in the same 401 response. (2) Implementation of a mass-DEREG anti-DDoS filter via a four-map architecture separating REGISTER and DEREG tracking. Tests using the raptor16 tool (multi-UE SIP simulator).
Assignment 4
POC "Mon Compagnon" — voice assistant — End-to-end voice assistant for phone call analysis. Pipeline: Whisper (speech-to-text) → Phi-3 MIT (summarisation/LLM) → SQLite (storage). Speaker diarisation added via pyannote.audio to distinguish Agent and Customer in transcriptions. Deployed on the "colossus" GPU server. Phi-3 selected for its MIT licence, compatible with patent filing.
Assignment 5
Cross-cutting research & technical watch — Inventory of annotated public PCAP datasets (NETRESEC, CIC/UNB, CAIDA, UNSW-NB15, Wireshark SampleCaptures, SIP/VoIP sources). Study of AI inter-agent protocols (MCP, Google A2A, IBM/BeeAI ACP) and their applicability to network infrastructure. Design of ML pipelines for TCP anomaly detection (slow start, congestion, bufferbloat). Training on transformer architectures (multi-head attention, scaling laws, nanochat/Karpathy).

Challenges

Challenge 01 · Network data
Maximum entropy of raw PCAP files
Raw PCAP files approach the theoretical entropy maximum (~7.5 bits/byte), saturating LLM context windows and severely degrading their comprehension. Solution: exploration of KPI extraction to CSV, dynamic entropy-based segmentation, and intermediate formats more compressible by models.
Challenge 02 · Fine-tuning
No variant outperforms the base model
Across 16 fine-tuning runs of Qwen2.5-3B, the best score (88/90) matches the base model. Systemic issues identified: insufficient training dataset quality, question repetition in certain prompts, hallucinations. Concrete realisation that fine-tuning requires structured evaluation and a high-quality dataset.
Challenge 03 · IPSEC/SIP
Confusion between two independent authentication systems
The A-SBC server can use MD5 for SIP digest authentication and SHA-1 for ESP IPSEC simultaneously in the same 401 response. Conflating these two levels caused subtle bugs (IPSEC not activated after SHA-1 migration). Solution: conditional logic explicitly separating the two systems in the Go code.
Challenge 04 · Infrastructure
Restricted network access in a corporate environment
Orange's corporate proxy blocks many external APIs (OpenRouter, direct HuggingFace). Adaptation: systematic use of Orange's internal LLM proxy (management.llmproxy.ai.orange) and proxy environment variable configuration for all outbound requests.
Challenge 05 · LLM evaluation
Subjectivity and reproducibility of metrics
Reproducibly evaluating LLM responses on telecom support questions is non-trivial. Developed a 5-criterion, 10-point rubric, with systematic scoring across 9 prompts × 17 models, enabling an objective ranking and actionable recommendations.
