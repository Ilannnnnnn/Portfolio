FitIA

AI-powered personalised fitness coaching application — RAG pipeline over 4M+ documents, LLM-generated structured training plans and progress tracking.

M2 Project · Team POC

Context

Existing fitness coaching applications offer generic programmes that are poorly tailored to individual needs. FitIA's goal was to build a system capable of generating truly personalised coaching advice and training plans by combining the power of LLMs with a structured fitness knowledge base — all accessible from a native mobile application.

Developed as a team project during the M2 year, FitIA is a fully functional Proof of Concept: a Flutter interface with a Material 3 dark theme, a Python FastAPI backend, and a RAG (Retrieval-Augmented Generation) pipeline based on Weaviate indexing over 4 million fitness documents. The application supports two operating modes — Standalone (direct Gemini API calls from Flutter) and Full RAG (FastAPI + Weaviate for context-grounded responses).

Architecture

The application relies on a three-layer architecture: the Flutter interface communicates via HTTP REST with the FastAPI server, which orchestrates semantic search in Weaviate and text generation via Gemini. SentenceTransformers generates the 384-dimensional embeddings injected into the vector database.

RAG Pipeline — How it works

The RAG pipeline is the heart of the Full RAG mode: it transforms a user question into a context-grounded response anchored in the fitness knowledge base.

Step 1
Question encoding — The user's question is converted into a 384-dimensional vector by SentenceTransformers (all-MiniLM-L6-v2), running locally on the FastAPI server.
Step 2
Near-vector semantic search — The vector is compared against 4M+ documents indexed in Weaviate via cosine similarity. The top-K=5 most relevant documents are retrieved.
Step 3
Context formatting by type — Retrieved documents are formatted according to their Weaviate class: exercise, exercise_qa or faq. Each type produces a distinct context block.
Step 4
User profile injection — Age, BMI, goals, level and medical conditions are systematically injected into the system prompt, making the AI coach aware of the user's constraints and contraindications.
Step 5
Conversation window — The last 10 messages of the conversation are included to maintain dialogue coherence, allowing the AI to reference previous exchanges and refine its recommendations.
Step 6
Gemini call with enriched prompt — The final prompt combines: fitness coach system prompt + RAG context + user profile + history. Gemini generates a Markdown response (chat) or structured JSON (training plan).
Step 7
Return to Flutter — The response is sent back to the mobile application via FastAPI. Markdown is rendered natively by flutter_markdown; JSON is parsed to display the structured training plan.

Technical highlight

Structured JSON generation via response_mime_type

For training plan generation, the Gemini API is called with response_mime_type: "application/json". This forces the model to return valid JSON rather than free text — no post-processing or fragile regex needed. The JSON schema (exercises, sets, reps, rest times, coaching notes) is described in the system prompt, and Gemini guarantees structural compliance of the response.

This pattern is combined with medical awareness: every prompt injects the user's declared health conditions (hypertension, diabetes, asthma…). Gemini automatically adapts the proposed exercises — reduced intensities, low-impact movements, integrated warnings in coaching notes — without any server-side filtering logic.

Fine-tuning dataset via Claude Opus

The knowledge base was enriched via an automated conversion script: raw dataset content (exercises, FAQs) is transformed into structured Q&A pairs via the Claude Opus API (Anthropic SDK). This fine-tuning pipeline produced documents better suited to semantic search and RAG context injection.
